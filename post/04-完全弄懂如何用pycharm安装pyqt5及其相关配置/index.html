配置专门的虚拟环境

原因：不同项目采用不同版本的python，所依赖的库的版本也不一样，为了避免版本冲突，所以给每一个项目每个python版本创建一个虚拟环境，环境中所使用的依赖库也是独立存在，不会被其他版本或其他项目的库影响。

单独创建一个文件夹来专门存放pyqt5的代码并建立虚拟环境

我在F盘的建了一个文件夹为python，在这个里面建立了一个文件夹python-pyqt5来专门存放代码，此时的python-pyqt5里面为空，没有任何文件

进入pycharm，并打开python-pyqt5项目

步骤：点击File–>点击Open–>找到刚才建立的文件夹（python-pyqt5）–>点击OK

配置虚拟环境

点击下方的Terminal,进入终端（必须确保终端上显示的路径与文件夹的路径一致）

在终端上输入

这个就是创建虚拟环境

1

python -m venv venv

此时在Project项目中python-pyqt5就可以看到一个文件夹为venv，这个就是虚拟环境

配置虚拟的python环境

步骤 ： 1、点击File–>Settings

2、找到Project:python-pyqt5–>Project Interpreter（项目解释器）–>Add（添加解释器）

3、点击Virtualenv Environment–>Existing environment–>Interpreter 这里的解释器要选择我们创建的虚拟解释器的路径，最后要包括python.exe

一路点击Ok回到Project Interpreter（项目解释器）,会发现现在已经创建好了解释器，此时这里就只有两个默认的库，最后先点击右下方的Apply,在点击OK。 这里的“（2）”不要管，这个是因为我已经创建了第二个这个的解释器。 如果是第一次创建，则这里就没有“（2）”

安装pyqt5和pyqt5-tools工具

在打开一个终端，这里要显示的venv,这样才算配置虚拟解释器成功，表示现在的环境是虚拟环境。

本次安装是使用的国内源，豆瓣网进行安装，安装速度会很快

安装:pyqt5

1

pip install pyqt5 -i https://pypi.douban.com/simple/

出现Successfully，即为安装成功，出现黄色的不用管，这个是警告，一般是告诉pip需要进行更新

安装:pyqt5-tools

1

pip install pyqt5-tools -i https://pypi.douban.com/simple/

配置系统环境变量

点击：控制面板右击–>点击系统–>高级系统设置–>系统变量–>Path

打开文件夹到虚拟环境的界面

在这里进行搜索

进行搜索：designer.exe

右击–>打开文件所在的位置

复制上面的路径：F:\python\python-pyqt5\venv\Lib\site-packages\qt5_applications\Qt\bin

添加到环境变量中

再次进行搜索：plugins

点击最后一个文件并右击–>进入到文件所在位置

点击plugins，进入该文件夹

复制路径：F:\python\python-pyqt5\venv\Lib\site-packages\PyQt5\Qt5\plugins

并添加到系统变量中

1
2

变量名：QT_QPA_PLATFORM_PLUGIN_PATH
变量值：路径

配置Qt Designer

进入pycharm,点击File–>点击Settings–>Tools–>External Tools

添加额外组件的名字

Name:

1

Qt Designer

额外组件的路径，这个路径就是designer.exe的路径

Program:：

1

F:\python\python-pyqt5\venv\Lib\site-packages\qt5_applications\Qt\bin\designer.exe

工作路径： 方法一： Working directory:

1

$FileDir$

方法二： Working directory:

1

F:\python\python-pyqt5

由于我进行配置为第一种是会报错，出现路径不存在的问题，因此我改为的第二种方法，填写一个实际的路径 进入初始界面，按照如下步骤进行点击

配置成功时，运行不会报错，并且会出现一个界面

出现这样的界面就代表Qt Designer配置成功
